# 유해 콘텐츠 탐지 모델 결과 보고서 (Version 1)

본 문서는 팀 내 비교 실험 결과를 기반으로 작성한 **1차 버전(Version 1)**이며,  
향후 임영재 모델을 동일 파이프라인(32프레임 균등 샘플링)으로 재학습/재평가한 결과가 도착하면  
해당 지표를 반영하여 **Version 2**로 업데이트될 예정이다.

---

# 1. 모델 성능 요약

## (1) 박상원 – 공개데이터(벤치마크)

- 이미지: Accuracy 98.44%, Precision 98.04%, Recall 99.68%, F1 98.86%
- 비디오: Accuracy 98.73%, Precision 98.29%, Recall 99.26%, F1 98.77%

## (2) 박상원 – 평가데이터(실제 수집 1200개)

- 이미지: Accuracy 67.06%, Precision 55.24%, Recall 92.31%, F1 69.12%
- 비디오: Accuracy 57.95%, Precision 54.35%, Recall 83.12%, F1 65.73%

## (3) 임영재 – 평가데이터(동일 평가셋)

※ 임영재 모델은 **박상원 파이프라인과 동일한 방식(32프레임 균등 샘플링)** 기준으로  
재학습 및 재평가가 예정되어 있으며, 아래 성능은 기존 방식으로 측정된 임시 결과임.

- 이미지: Accuracy 67.77%, Precision 57.28%, Recall 75.64%, F1 65.20%
- 비디오: Accuracy 71.02%, Precision 66.23%, Recall 82.14%, F1 73.30%

## (4) 안지산 – 평가데이터(동일 평가셋, 비디오 단일 모델)

- Default threshold 기준: F1 5.83%
- Best threshold 기준: F1 56.49%

→ 성능이 충분히 확보되지 않아 최종 후보에서 제외함.

---

# 2. 최종 모델 선정 기준 및 결과

동일 평가데이터(팀원 3명이 자체 수집한 이미지·비디오)를 기반으로 비교한 결과:

- 이미지 분야: **박상원 모델이 가장 우수(F1 기준)**
- 비디오 분야: **임영재 모델이 가장 우수(F1 기준)**

이에 따라 **최종 파이프라인 구성(Version 1)**은 다음과 같이 결정하였다.

- 이미지 입력 → 박상원 이미지 모델 사용
- 비디오 입력 → 임영재 비디오 모델 사용(동일 샘플링 기준으로 재평가 후 Version 2 반영)

임영재 모델은 공정한 비교를 위해 다음을 진행 중이다.

- 32프레임 균등 샘플링 방식으로 재학습
- 공개데이터·평가데이터 기준 재평가
- 동일 포맷으로 성능지표 제공 예정

현재 문서는 박상원 기준 지표 중심으로 정리된 **초기 버전(Version 1)**이다.

- 일관된 라벨링 및 평가 기준 확립.

- 이미지·비디오 파일럿 테스트 수행 후 구조 적합성 검증.
- 멀티모달 구조 가능성 분석.

### 3. 실제 데이터 수집 - 충족

- 폭력·음주·흡연·위험행동 위주 실제 사례 수집.

- YOLO(객체) + CLIP(문맥) + SlowFast(행동) + 규칙 기반의 멀티모달 구조 개발.
- 벤치마크 기준 정확도:
  - 이미지: Accuracy 98.44%, F1 98.86%
  - 비디오: Accuracy 98.73%, F1 98.77%

## 도전 과제 충족 여부

### 1. 벤치마크 기준 정확도 0.80 이상 - 충족
- 이미지·비디오 모두 Accuracy 0.98 이상으로 목표를 초과 달성.

### 2. 모델 경량화 - 부분 충족

충족한 사항  
- 입력 특징을 537/937차원 → 256차원으로 축소하여 연산량·메모리 사용량 감소.  
- YOLOv8n, CLIP-B/32 등 비교적 경량한 모델을 채택.

미충족한 사항  
- Distillation, Pruning, Quantization, TensorRT/ONNX 등 공식적인 경량화 기법은 적용하지 않음.  
- Backbone 자체의 경량화는 진행되지 않음.

결론  
- 구조적 수준의 경량화는 있으나 전문적인 경량화 연구는 아니므로 “부분 충족”으로 판단.

### 3. 설명가능한 모델(XAI) - 부분 충족

충족한 사항  
- 객체(YOLO), 문맥(CLIP), 행동(SlowFast) 모듈 분리가 가능해 판단 근거가 구조적으로 해석 가능.  
- 객체 기반 규칙(infer_behavior_from_objects)을 통해 예측 근거를 사람이 확인 가능.

미충족한 사항  
- Grad-CAM, SHAP, Attention Map 등 정식 XAI 시각화 기능 미제공.  
- 내부 의사결정을 heatmap으로 표현하는 기능은 구현되지 않음.

결론  
- 구조적 설명가능성은 확보했으나 XAI 구현 관점에서는 “부분 충족”.
